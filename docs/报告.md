# AgriWorld 项目阶段汇报

> 汇报对象：导师组  
> 当前主线：`agriworld-v3.23`  
> 数据版本：2019-2023 五年合并数据  
> 评估协议：2019-2022 训练，2023 时间外推验证

## 1. 项目目标

AgriWorld 的目标是构建一个面向县域农业系统的物理引导可微世界模型。模型不只追求产量预测精度，还希望能够解释和审计农业关键因素对作物生长的影响，例如降水、辐射、VPD、氮素和极端高温。

当前模型以作物生长机理为骨架，结合小型神经专家模块，模拟 LAI、生物量、土壤氮库和土壤水分的日尺度动态，并输出最终产量。

![模型工作流](report_assets/02_model_workflow.png)

## 2. 当前数据进展

当前版本已经从早期单年或小样本调试，推进到 2019-2023 五年合并数据。经过天气、作物类型、LAI 覆盖度和产量标签等质量控制后，当前用于主线训练与验证的 corn 子集为 867 个 county-year 样本。

![数据质控漏斗](report_assets/01_data_funnel.png)

当前划分方式：

| 数据部分 | 年份 | 样本数 |
|---|---|---:|
| Train | 2019, 2020, 2021, 2022 | 680 |
| Validation | 2023 | 187 |

这个设置对应时间外推验证，能更好检查模型是否具备跨年份泛化能力，而不是只在随机划分下拟合局部统计规律。

## 3. 模型版本进展

当前主线为 `agriworld-v3.23`。相比早期版本，主要改进包括：

1. 数据管线修复：解决了历史数据中 Tmean 全零、GDD 错误和有效样本不足的问题。
2. 训练效率优化：数据集加载到内存，默认日尺度 Euler/stride 求解，显著缩短训练时间。
3. 物理约束增强：加入状态边界、水分下限、冠层-产量一致性和参数先验。
4. 温度机制重构：放弃过强的硬温度胁迫，改为极端高温门控。
5. 因素响应审计：新增对降水、辐射、VPD、氮素、温度和极端高温的反事实测试。
6. 工程结构整理：核心代码迁入 `agriworld/` 包，训练评估脚本迁入 `scripts/`。

当前温度机制的核心思想是：正常暖日主要通过 GDD 影响物候，不再被直接当作胁迫；只有当温度进入极端高温区间时，才引入轻量负效应。

## 4. 当前评估结果

当前主线 baseline 在 2023 验证集上的产量表现如下：

![核心指标](report_assets/03_metric_cards.png)

| 指标 | 数值 |
|---|---:|
| 验证样本 | 187 |
| RMSE | 21.53 bu/ac |
| NRMSE | 11.14% |
| Corn MAPE | 8.71% |
| R2 | -0.298 |

从误差量级看，当前模型已经进入可用区间；但 R2 仍为负，说明模型对 2023 年不同县域之间的相对排序能力还不足。换句话说，模型已经能较好把握总体产量水平，但对空间异质性的表达仍需加强。

## 5. 农业因素响应

AgriWorld 的一个重要目标是“响应方向正确”，而不是只在产量标签上取得较低误差。当前因素响应审计结果如下：

![因素响应](report_assets/04_factor_response.png)

| 因素 | high-minus-low yield response | 当前解释 |
|---|---:|---|
| 降水 | +1.97% | 响应较弱，可能因为当前高产玉米带水分限制不强 |
| 辐射 | +49.13% | 正向，符合光合驱动逻辑 |
| VPD | -29.34% | 负向，符合高蒸汽压亏缺抑制作物生长的预期 |
| 氮素 | +30.46% | 正向，说明氮素响应已经恢复 |
| 温度 | -0.36% | 接近中性，符合当前 soft/heat 设计 |
| 极端高温 | -0.22% | 负向但偏弱，可继续调优 |

总体看，辐射、VPD 和氮素响应已经比较清晰；降水和极端高温响应仍偏弱，是后续优化重点。

## 6. 消融实验结论

当前关键消融显示，极端高温门控版本是目前更稳的主线：

![消融结果](report_assets/05_ablation_rmse.png)

主要结论：

1. `baseline` 优于关闭温度胁迫的版本，且保留了极端热害负响应。
2. `heat_stress_025` 的热害响应更强，但产量误差略差，因此不作为主线。
3. `hard_temperature_stress` 显著恶化，说明硬温度胁迫会过度压低产量。
4. `no_water_floor` 在历史完整消融中崩溃，说明土壤水分下限是必要物理约束。
5. `no_vpd_stress` 即使产量误差接近，也会失去 VPD 负响应，物理上不可接受。

因此，当前主线应继续保留：

- 极端高温门控；
- VPD stress；
- nitrogen stress；
- water floor；
- year trend；
- canopy loss；
- CouplingHead。

## 7. 工程整理进展

当前项目目录已经完成初步工程化整理：

```text
AgriWorld/
├── agriworld/      # 核心模型包
├── scripts/        # 训练、评估、数据管线入口
├── docs/           # 技术文档和汇报材料
├── results/        # 评估与消融结果
├── saved_models/   # checkpoint
├── benchmarks/     # 对照基线
└── tests/          # 单元测试
```

旧命令仍兼容，例如：

```bash
python train.py
python evaluate.py
python factor_response.py
python ablation.py
```

真实实现则位于：

```bash
python -m scripts.train
python -m scripts.evaluate
python -m scripts.factor_response
python -m scripts.ablation
```

## 8. 当前问题与风险

当前版本的主要问题不是训练失败，而是模型表达能力进入更细的瓶颈：

1. 县域间排序能力不足：R2 仍为负，说明空间异质性没有充分学到。
2. 水分状态验证较弱：SMAP probe 表明 ODE soil-water 状态与遥感土壤水分仍不够一致。
3. 降水响应偏弱：可能与高产玉米带水分非限制、土壤参数、ET/drainage 设置有关。
4. 极端高温响应偏弱：方向正确，但幅度可能不足。
5. 因素响应 PASS 判据仍需收紧，避免近零响应被过度解释为通过。

## 9. 下一步计划

![下一步路线](report_assets/06_next_steps.png)

建议下一阶段按以下顺序推进：

1. **修严因素响应审计**  
   收紧 `scripts/factor_response.py` 中正/负响应 PASS 判据，确保报告中的机制结论更可靠。

2. **增强空间异质性**  
   增加由静态特征调制的 yield residual 或 interaction coefficients，让模型能区分不同县域的产量差异。

3. **改进水分状态**  
   对齐 ODE soil water 与 SMAP 的深度、尺度和时间聚合方式，检查 root depth、ET 和 drainage 参数。

4. **固定论文实验协议**  
   形成稳定实验矩阵：baseline、no temperature、strong heat、no water floor、no VPD、no nitrogen、no year trend。

5. **准备论文叙事**  
   以“物理约束 + 可微世界模型 + 农业因素反事实响应”为主线，突出模型不是黑箱预测器，而是可解释、可审计、可用于农业情景推演的世界模型。
